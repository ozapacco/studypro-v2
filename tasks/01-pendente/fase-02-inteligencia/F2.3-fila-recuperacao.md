# F2.3: Fila de Recuperação

**Skill**: Planner Engine + Topic Engine + Backend  
**Estimativa**: 8h  
**Prioridade**: P0  
**Dependências**: F1.4, F2.2

## Descrição
A fila de recuperação (recoveryQueue) é o backlog de tópicos e matérias que exigem atenção imediata. Os itens entram nesta fila com base no acúmulo de erros frequentes e são reciclados de volta para estarem nas revisões do usuário até saírem do status crítico.

## Subtasks
- [ ] F2.3.1 Criar tabela `recoveryQueue` no banco de dados
- [ ] F2.3.2 Implementar gatilhos automáticos de entrada (Edge Functions ou Listeners de sessão)
  - [ ] erro repetido em múltiplas sessões num intervalo curto
  - [ ] erro crônico mapeado em simulados recentes
  - [ ] taxa despencando rapidamente (taxa muito baixa num tópico de alta utilidade)
  - [ ] erro marcado via botão de feedback manual ("nunca aprendi direito")
- [ ] F2.3.3 Implementar a sugestão de ações corretivas (revisar teoria rápida, refazer x questões resolvidas, criar mais cards)
- [ ] F2.3.4 Criar status na respectiva entidade: `open`, `in_progress`, `done`, `archived`
- [ ] F2.3.5 Cron / task para rechecagem da fila de prioridade em 48-72h
- [ ] F2.3.6 Condição de saída (marcar com melhora mínima provada via acertos na recuperação)
